CI/CD Automation for Portfolio

Using GitLab CI/CD, Docker, and Ansible on a Private Server

This project aimed to automate the complete deployment lifecycle of my Dockerized portfolio website using GitLab CI/CD. Unlike cloud platforms such as AWS, this deployment was executed on a private server, adding complexity and unique challenges.

Pipeline Workflow

  • Build the Docker image and tag it using GitLab CI
  • Push the image to GitLab's Container Registry
  • Connect to the private server using SSH
  • Deploy the Docker container using Ansible and Bash scripts
  • Perform post-deployment health checks

Challenges & Fixes

1. SSH Permission Denied

Cause: SSH key had insecure file permissions in the pipeline environment.

Fix: Used chmod 600 on the private key before connection.

2. Docker Daemon Not Reachable

Cause: GitLab runner couldnâ€™t communicate with the Docker daemon.

Fix: Disabled TLS using DOCKER_TLS_CERTDIR="" and set DOCKER_HOST appropriately.

3. Ansible Extra Vars Failure

Cause: Incorrect usage of --extra-vars syntax.

Fix: Corrected command formatting and removed an unnecessary backslash.

4. Healthcheck Curl Not Found

Cause: The curl package was not pre-installed in the job image.

Fix: Installed it using apk add --no-cache curl before the check.

Final Output

  • Fully automated deployment pipeline using GitLab CI/CD
  • Containerized portfolio deployed on a private Linux server
  • Integrated healthcheck to verify successful deployment
  • Pipeline triggered on every push to the main branch
